# 常见问题

## 构建问题

### NSIS 安装包下载失败

**问题**：执行 `pnpm tauri build` 时，NSIS 下载失败，提示 TLS 证书错误：

```
Error failed to verify TLS certificate: UnknownIssuer
```

**原因**：网络环境中的代理或防火墙导致 TLS 证书验证失败。

**解决方案 1（推荐）**：只生成 MSI 安装包

修改 `src-tauri/tauri.conf.json`：

```json
{
  "bundle": {
    "targets": ["msi"]
  }
}
```

然后执行：

```sh
pnpm tauri build
```

**解决方案 2**：跳过 NSIS 下载

设置环境变量：

```sh
set TAURI_SKIP_NSIS_DOWNLOAD=1
pnpm tauri build
```

**解决方案 3**：手动下载 NSIS 并配置

1. 手动下载 NSIS：https://github.com/tauri-apps/binary-releases/releases/download/nsis-3.11/nsis-3.11.zip
2. 解压到项目根目录的 `tools/nsis-3.11/`
3. 将 NSIS 目录添加到系统 PATH 环境变量

**备注**：

- exe 文件仍然可以正常使用，构建产物位于 `src-tauri/target/release/tauri-app.exe`
- MSI 安装包位于 `src-tauri/target/release/bundle/msi/`
